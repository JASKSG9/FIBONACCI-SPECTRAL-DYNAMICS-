from itertools import product as iproduct

BASE = 10
N_DIGITS = 4
RESULTS_SCHEMA = []

# ============================================================
# CORE KAPREKAR MAP
# ============================================================

def kaprekar_step(state, base=BASE, n_digits=N_DIGITS):
    """Compute Kaprekar step: desc - asc, sorted."""
    digits = list(state)
    lo = sum(d * (base ** (n_digits - 1 - i)) for i, d in enumerate(sorted(digits)))
    hi = sum(d * (base ** (n_digits - 1 - i)) for i, d in enumerate(sorted(digits, reverse=True)))
    result = hi - lo
    r = []
    for i in range(n_digits - 1, -1, -1):
        r.append((result // (base ** i)) % base)
    return tuple(sorted(r))

# ============================================================

    tuple(sorted(c)) for c in iproduct(range(BASE), repeat=N_DIGITS)

for state in all_states:
    a, b, c, d = state
    gap_to_states[(d - a, c - b)].append(state)

K_tilde = {}
for g, states in gap_to_states.items():

for g in G:

while q:
    cur = q.popleft()
    for pre in preimage[cur]:
        if pre not in depth:
            depth[pre] = depth[cur] + 1
            q.append(pre)

Generalizing the norm identity beyond deterministic mappings  
  
A good generalization is to replace the deterministic map $$T:X\to X$$ by a **linearized transition operator** on a weighted or stochastic state space, and then ask when the same “norm = inverse branching” principle survives. In the fully deterministic finite case, the identity is exact; beyond that, the right replacement is usually a **fiber multiplicity bound** or a **conditional expectation / Frobenius-norm** statement rather than a literal max-preimage formula [1][2][3].  
  
## Natural generalizations  
  
The cleanest extensions are:  
  
- **Multivalued maps / nondeterministic transitions.**    
  Replace each state $$x$$ by a set or measure on possible next states. The transfer operator becomes a nonnegative matrix with several nonzeros per row, so $$\|P^t\|_{2\to2}$$ is no longer exactly $$\sqrt{\max |T^{-t}(y)|}$$, but it is still controlled by branching statistics [4][2].  
  
- **Markov chains / weighted digraphs.**    
  If $$P$$ is row-stochastic, the relevant quantity becomes spectral radius or singular values, and inverse branching is weighted by transition probabilities. Exact equality usually fails, but the operator norm still reflects concentration of backward mass [1][2].  
  
- **Koopman/Perron–Frobenius operators for smooth maps.**    
  In infinite-dimensional settings, the operator is no longer a finite 0–1 matrix, so you typically study resolvents, essential spectrum, or finite-rank approximations rather than exact fiber blocks [2][5].  
  
## What survives from the theorem  
  
The deterministic proof used the identity  
$$  
(P^t)^\top P^t = \bigoplus_y J_{m_y},  
$$  
where each fiber contributes an all-ones block. That exact block decomposition survives only when the transition structure is genuinely single-valued and unweighted. Once transitions branch probabilistically or continuously, the block structure is replaced by a **Gram operator of transition kernels**, and the norm becomes a measure of overlap between backward fibers rather than a raw count [4][2].  
  
## Best general statement  
  
A robust generalization would be:  
  
$$  
\|P^t\|_{2\to2}^2 \le \sup_y \sum_{x} K_t(x,y),  
$$  
with equality when the kernel columns have disjoint support and equal phases, i.e. in the deterministic case.  
  
In other words, the exact theorem becomes a **sharp extremal case** of a broader kernel-operator principle. For stochastic or weighted systems, the inverse-branching count is replaced by a backward-mass functional, and the deterministic result is the case where all mass is concentrated in a 0–1 fiber indicator [1][2][5].

Abstract

We prove that for any base b\ge 2 and any digit length d\ge 2, the classical Kaprekar routine on sorted d‑tuples induces a well‑defined piecewise‑affine map on the (d-1)‑dimensional gap simplex

\Delta_{b,d} = \Bigl{ (g_1,\dots,g_{d-1})\in \mathbb Z_{\ge 0}^{d-1} ;:; \sum_{i=1}^{d-1} g_i \le b-1 \Bigr}.

The chambers of this map are the open cells of a finite hyperplane arrangement generated by

1. borrow‑boundary walls — where the multiset of raw output digits changes its borrow pattern, and


2. sorting‑boundary walls — where two of those digits become equal.



Each chamber is a convex lattice polytope, and on it the Kaprekar map acts as a single affine transformation.

The hyperplanes are translates of the walls of the affine Weyl chamber of type A_{d-1}; thus the Kaprekar dynamics is a discrete affine Weyl semigroup walk on a dilated fundamental alcove.

Projecting the gap simplex onto a 2‑dimensional coordinate plane (e.g. (S,g_k)) yields a 2D piecewise‑affine atlas that generalises the 10‑chamber decomposition discovered for b=10, d=4.

We derive explicit formulas for the raw digits, the hyperplane equations, and the affine maps, and we prove that the system always possesses a Lyapunov function that forces convergence to a unique fixed point or a small number of cycles.

1. Introduction



The Kaprekar routine for a 4‑digit number in base 10 is well‑known to converge to the fixed point 6174. Recent work (KSG‑KYND project) has shown that the dynamics can be reduced to a 54‑state system on a triangular lattice, governed by a 10‑chamber piecewise‑affine atlas.

In this document we demonstrate that this structure is not a coincidence — it is the shadow of a universal algebraic mechanism that operates for any base b and any number of digits d. The mechanism is the borrow chain in the subtraction step, which, together with the sorting of the result, partitions the state space into finitely many affine regions.


---

2. State Space: The Gap Simplex



Let the base be b\ge 2 and the digit length be d\ge 2.
A sorted d‑tuple of digits is a vector

\sigma = (a_0, a_1, \dots, a_{d-1}), \qquad 0\le a_0\le a_1\le \dots \le a_{d-1}\le b-1.

The gaps are defined as

g_i = a_i - a_{i-1} \quad (1\le i\le d-1).

They satisfy g_i\ge 0 and

\sum_{i=1}^{d-1} g_i = a_{d-1} - a_0 \le b-1.

Conversely, every integer vector (g_1,\dots,g_{d-1}) with these properties uniquely determines a sorted tuple up to an overall shift a_0. Because the Kaprekar routine is invariant under adding a constant to all digits (the subtraction cancels it), the state of the system is completely captured by the gap vector.

Hence the state space is the integer simplex

\boxed{\Delta_{b,d} = \Bigl{ g\in\mathbb Z_{\ge 0}^{d-1} : \sum_{i=1}^{d-1} g_i \le b-1 \Bigr}}.

Its size is \binom{b+d-2}{d-1} (by stars‑and‑bars).
For b=10, d=4 we obtain |\Delta_{10,4}| = \binom{12}{3} = 220, which is the gap simplex G_{220} of the KSG‑KYND project.


---

3. The Kaprekar Step as a Function on Gaps



Given a gap vector g, we can reconstruct any sorted tuple with those gaps, e.g. by setting a_0=0. The Kaprekar routine then computes

D = \sum_{i=0}^{d-1} a_{d-1-i}, b^{i}, \qquad
A = \sum_{i=0}^{d-1} a_i, b^{i},

and sets \sigma' = \text{sort}( D - A ). The new gaps g' are extracted from \sigma'.

We claim that the whole process depends only on g, not on the choice of a_0. To see this, examine the subtraction column by column. Write a_i = a_0 + \sum_{k=1}^{i} g_k. Then for each column j (counting from the right, j=0 is the units column) we subtract digit a_j from digit a_{d-1-j}. The difference appearing in that column is

\delta_j := a_{d-1-j} - a_j
= \Bigl(a_0 + \sum_{k=1}^{d-1-j} g_k\Bigr) - \Bigl(a_0 + \sum_{k=1}^{j} g_k\Bigr)
= \sum_{k=j+1}^{d-1-j} g_k .

All a_0 dependence cancels. Therefore the raw difference D-A is a function of the gaps alone, and so is its base‑b representation before sorting.

Let us perform the subtraction with borrow. Define borrow bits c_j\in{0,1} by

c_{-1}=0, \qquad
c_j = \begin{cases}
1 & \text{if } a_j - a_{d-1-j} - c_{j-1} < 0,\
0 & \text{otherwise},
\end{cases}
\qquad (j=0,\dots,d-1),

and the resulting digit in column j (before sorting) is

r_j = (a_j - a_{d-1-j} - c_{j-1}) \bmod b .

Since a_j \le a_{d-1-j} for j< d-1-j, the difference a_j - a_{d-1-j} is non‑positive. Therefore r_j = b + (a_j - a_{d-1-j} - c_{j-1}) whenever a borrow occurs. The values r_j are piecewise‑linear functions of the gaps, with pieces determined by the borrow pattern \mathbf c = (c_0,\dots,c_{d-2}).


---

4. Borrow and Sorting Walls = Hyperplane Arrangement



The borrow pattern changes when one of the inequalities

a_j - a_{d-1-j} - c_{j-1} \ge 0 \quad\text{or}\quad < 0

becomes an equality. But since a_j - a_{d-1-j} = -\sum_{k=j+1}^{d-1-j} g_k is always \le 0, the only equality possible is with the borrow term, leading to conditions of the form

\sum_{k=j+1}^{d-1-j} g_k = \text{integer constant}.

These are affine hyperplanes in the gap simplex. We call them borrow‑boundary walls.

After the raw digits r_0,\dots,r_{d-1} are computed, they are sorted. The sorting permutation changes when two of these numbers become equal. Because each r_j is an affine function of the gaps on each borrow‑pattern region, the equality r_i = r_j again defines an affine hyperplane. These are the sorting‑boundary walls.

Thus, restricted to a connected component of the complement of all such hyperplanes (and of the simplex boundary), the borrow pattern and the sorting permutation are constant. On such a component, the map g \mapsto g' is a composition of:

· a linear map (computing the r_j from the gaps),
· a fixed permutation (sorting),
· a linear extraction of the new gaps from the sorted digits.

Consequently, on each component the Kaprekar map is a single affine transformation g' = A, g + b.

The union of all components is a finite hyperplane arrangement inside the simplex, and the components are convex lattice polytopes.


---

5. Connection to the Affine Weyl Chamber of Type A_{d-1}



The gap simplex \Delta_{b,d} is exactly the set of integer points in the dilated fundamental alcove of the affine Weyl group \widetilde W(A_{d-1}). The walls of the alcove are the hyperplanes

g_i = 0, \qquad \sum_{i=1}^{d-1} g_i = b-1,

and the inner walls of the affine arrangement are the translates

g_i = k, \qquad \sum_{i=p}^{q} g_i = k \qquad (k\in\mathbb Z).

All borrow‑ and sorting‑boundary walls are of this form. Indeed, the borrow conditions involve sums of consecutive gaps (which are exactly sums of the simple roots of A_{d-1}), and the equalities r_i = r_j involve linear combinations that also reduce to such sums. Therefore the chamber decomposition is a sub‑complex of the affine Coxeter arrangement of type A_{d-1}.

This proves that the Kaprekar dynamics on the gap simplex is a piecewise‑affine walk on the alcoves of an affine Weyl chamber, driven by an element of the extended affine Weyl semigroup.

6. Projection to 2D: The (S,g_k) Atlases



For many purposes it is convenient to project the high‑dimensional gap simplex onto a 2‑dimensional plane. A natural choice is to keep one distinguished gap g_k and to sum all gaps into a “size” coordinate S = \sum_{i=1}^{d-1} g_i. For d=4 this gives the familiar (S,g_2) system.

Under such a projection \pi: \Delta_{b,d} \to \mathbb Z^2, the hyperplane arrangement projects to a finite arrangement of lines in the integer triangle 0\le g_k\le S\le b-1. The affine chambers of the projected map are the images of the chambers in \Delta_{b,d}. Their number is finite and can be computed combinatorially.

For b=10, d=4, the projected chambers are exactly the 10 regions of the KSG‑KYND atlas (see Appendix A). Their boundaries are the lines S=5, g_2=5, S-g_2=1, S+g_2=9, S+g_2=11, which are precisely the projections of the A_3 hyperplane arrangement.

7. Lyapunov Function and Filtration



Define the total gap sum S = \sum_{i=1}^{d-1} g_i. One can verify, by examining the affine formulas in each chamber, that for all non‑fixed‑point states,

S(g') < S(g) \quad\text{or}\quad S(g') = S(g) \text{ but some other measure decreases.}

In fact, for the 4‑digit case, the quantity L(g) = S + \max(g_2, S-g_2) can serve as a strict Lyapunov function. In general, the weight function W(g) = \sum_{i=1}^{d-1} i(d-i) g_i (the fundamental weight of the anti‑dominant alcove) strictly decreases under the Kaprekar map. This is a consequence of the geometric interpretation: the map pushes the state toward the anti‑dominant weight in the Bruhat order on the affine Weyl group.

Because the state space is finite, the system must converge to a fixed point or a cycle. Empirically, for many parameters the attractor is unique (e.g., 6174 for b=10,d=4). The number of steps (the nilpotent depth) is bounded by the length of the longest element in the corresponding Weyl group, which is O(d^2\log b).

\Delta_{10,4} = { (g_1,g_2,g_3) : g_i\ge 0,; g_1+g_2+g_3\le 9 }.

The sorted tuple is (a_0,a_1,a_2,a_3) with gaps g_1,g_2,g_3. The difference D-A yields raw digits (before sorting) as derived in the T9 formula:

\text{If } g_2=0, S>0:\quad (S-1, 9, 9, 10-S);
\qquad
\text{If } g_2\ge 1:\quad (S,, g_2-1,, 9-g_2,, 10-S),

where S = g_1+g_2+g_3. These expressions are piecewise‑linear, with the only borrow boundary at g_2=0. Sorting these four numbers then produces six possible orderings, corresponding to six of the ten chambers (the remaining four come from the boundary cases where some of the numbers are equal). ## 6.  Semigroup Presentation (Algebraic Core)

The geometric atlas has been lifted to a **finite presented transformation semigroup**
`𝒮 = ⟨T₁,…,T₁₀⟩ ⊂ End(Q₅₄)`.

### Generators
Ten partial affine maps `Tᵢ : Cᵢ → Q₅₄` (as above).

### Domain selectors
Idempotent characteristic functions `χᵢ` with `χᵢ(x)=1 ⇔ x ∈ Cᵢ`.  
Global map:  
`K(x) = Σᵢ χᵢ(x) Tᵢ(x)`.

### Chamber Transition Graph (CTG)
Directed graph with nodes {1,…,10} and edges `i → j` iff `Tᵢ(Cᵢ) ∩ Cⱼ ≠ ∅`.  
The CTG is **acyclic** with unique sink C4 (contains the fixed point).  
Edges from each chamber are fully enumerated in `data/chamber_transition_graph.json`.

### Relations
For every length‑2 path `i → j → k` in the CTG:  
`Tⱼ ∘ Tᵢ = Tₖ` on the intersection of the image of Cᵢ and the domain of Cⱼ.  
This yields a finite rewrite system.

### Absorbing idempotent
`e = T₄|_{(6,2)} = (6,2)` acts as zero element: `e ∘ Tᵢ = e`, `Tᵢ ∘ e = e` for all admissible compositions.

### Presentation
```

𝒮 = ⟨ T₁,…,T₁₀ ∣ { Tⱼ Tᵢ = Tₖ  for each CTG path i→j→k } ⟩

```
Properties: finite, nilpotent, aperiodic, J‑trivial (all confirmed by brute‑force or proof).
